【为什么之前“全是乱码”，怎么修的】
  引擎是 2005 年的 Shift-JIS 日文引擎。仅改字体字符集不够，正文画字函数
  (bodyPump @0x446E50) 用一张写死的 Shift-JIS 字节分类表 (0x48CAE8) 按首字节高半字节
  查表，把 GBK 汉字首字节 0xA0–0xDF（绝大多数汉字都在此区）误判成单字节 →
  逐字节按 Latin-1 画出 → 乱码。
  关键修复：把分类表 0xA0–0xDF 四项由“单字节”改“双字节”，并把 ≥0xEB 的字节顺序
  分支强制走正确路径；再配合字体字符集 GB2312、宋体、以及 DBCS 前导字节判定
  (IsSJISLead) 扩到 GBK 全区 —— 正文即正常显示中文。

【做了什么（补丁清单）】
  exe 二进制补丁（均已校验原始字节、等长不改大小）：
    · 正文渲染修复（分类表 4 字节 + 顺序分支 1 字节）—— 核心去乱码
    · DBCS 前导字节判定 IsSJISLead 扩到 0x81–0xFE
    · 字体 lfCharSet SHIFT_JIS→GB2312（两处）、字体名→宋体（两处）
    · 去“不支持的OS”警告框（两个分支）
    · 去 CD-DA 音乐提示框
    · voice 资源校验补丁
  exe 界面字符串：约 110 条菜单/存读档/设置/报错/确认框日文 → 中文（等长 GBK 回填）
  注册表：GrInstall/MusicInstall/VoiceInstall/Music=1 → 游戏读本地 .lax，不再找光盘
          （否则会弹“CD驱动器错误/找不到文件”要求插盘）；Font=宋体
  scenario.lax：全剧本日→中。等长 GBK 回填保持 VM 偏移不变；多代理协同翻译，
          补齐了原来未译/半译/含非GBK符号(♪★等)的行；剩余极少数未译行已做
          Shift-JIS→GBK 等长转换，显示为“可读日文”而非乱码；未命名角色名
          (？？？) 也已修正。
